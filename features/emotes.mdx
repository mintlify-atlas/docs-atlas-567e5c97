## How FFZ Emotes Work

FFZ emotes are stored in **emote sets**. When a user visits a channel, FFZ loads the relevant sets from the FFZ API and injects their CSS into the page via a `ManagedStyle` element keyed by set ID (e.g. `es--12345`). Each emote in a set carries an ID, name, and image URLs at 1Ă—, 2Ă—, and 4Ă— densities.

Emote source priorities are configurable through the `chat.emotes.source-priorities` setting, which exposes a drag-and-drop UI listing all registered providers.

### Emote Sets

FFZ tracks three categories of sets:

| Set Type | Description | Internal Collection |
|---|---|---|
| **Global sets** | Available to every user on every channel | `emotes.global_sets` |
| **Default sets** | Activated by a provider for a specific channel or context | `emotes.default_sets` |
| **Sub sets** | Granted to subscribers or specific users | `emotes.sub_sets` |

The FFZ provider (`ffz`) is always registered. Add-ons can register additional providers via `emotes.setProvider()` and load set data via `emotes.addDefaultSet()` / `emotes.addSubSet()` / `emotes.loadSetData()`.

### FFZ Emote Server Integration

FFZ communicates with the FFZ API over PubSub to receive real-time emote changes. When the server sends `pubsub:command:add_emote` or `pubsub:command:follow_sets`, the local emote set is updated immediately without a page reload.

## Modifier Emotes

Modifier emotes are special emotes that apply visual effects to the emote immediately preceding them in chat. They must be placed **after** a base emote with no extra text between them. When the tokenizer detects a modifier emote following a compatible base emote, both are wrapped in a `modified-emote` container with a `data-effects` attribute listing the active modifier flag bits.

```html
<!-- Example: PogChamp followed by a Rainbow modifier -->
<span class="modified-emote" data-effects="2048">
  <img class="chat-image chat-line__message--emote" ...>
  <!-- modifier emote rendered inline -->
</span>
```

The CSS for each effect is generated at runtime and injected via a separate `ManagedStyle('effects')` instance.

<Note>
Combining `Appear` + `Leave` produces an in-out animation (`ffz-effect-in-out`) over 6 s instead of playing both animations independently. Combining `Bounce` + `FlipY` adds a `translateY(100%)` transform so the bounce originates from the top.
</Note>

Animated effects depend on the `chat.emotes.animated` setting being enabled. Each animated effect has its own settings key (e.g. `chat.effects.Slide`, `chat.effects.Rotate`) so individual effects can be disabled even when global animation is on.

## The Emote Menu

FFZ adds a custom emote picker to chat. It lists all emotes from every active provider, organized by provider icon and set name. Emotes can be marked as favorites per-provider (e.g. `ffz`, `twitch`) and are stored in local settings.

## Settings Reference

| Setting Key | Default | Description |
|---|---|---|
| `chat.emotes.enabled` | `2` (Enabled) | `0` = disabled, `1` = Twitch only, `2` = all emotes |
| `chat.emotes.2x` | `0` | `0` = normal, `1` = larger emotes, `2` = larger emotes and emoji |
| `chat.emotes.limit-size` | `true` | Cap native Twitch emote size to prevent oversized emotes |
| `chat.emotes.allow-gigantify` | `true` | Allow the "Gigantify an Emote" Channel Points Power-Up |
| `chat.fix-bad-emotes` | `true` | Clean up white borders / solid backgrounds on bad global emotes |
| `chat.click-emotes` | `true` | Open emote info pages by Shift-clicking emotes |
| `chat.sub-emotes` | `true` | Open Twitch sub pages by Shift-clicking sub emotes |
| `chat.emote-dialogs` | `true` | Open emote info cards for Twitch emotes by clicking |
| `chat.emotes.source-priorities` | `null` | Provider priority order (drag-and-drop UI) |
| `chat.effects.enable` | `true` | Master toggle for all emote effects |
| `chat.effects.<Flag>` | varies | Per-effect toggle (e.g. `chat.effects.Rotate`, `chat.effects.Rainbow`) |
