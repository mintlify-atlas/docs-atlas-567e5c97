This guide walks you through setting up the FrankerFaceZ development environment and creating a minimal working add-on.

## Prerequisites

- [Node.js](https://nodejs.org/) (LTS recommended)
- [pnpm](https://pnpm.io/) package manager

## Set up the development environment

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/FrankerFaceZ/FrankerFaceZ.git
    cd FrankerFaceZ
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    pnpm start
    ```

    This starts `webpack-dev-server` on port `8000`. The server watches for file changes and rebuilds automatically.

    <Warning>
      The development server uses a self-signed HTTPS certificate. You need to accept this certificate in your browser before the dev server will work. Visit `https://localhost:8000` directly to accept it.
    </Warning>
  </Step>

  <Step title="Switch to the userscript version on Twitch">
    For development, use the FrankerFaceZ userscript version rather than the browser extension. The userscript is designed to load from your local development server automatically.

    Install a userscript manager (Tampermonkey or Greasemonkey) and install the FrankerFaceZ userscript.
  </Step>

  <Step title="Enable debug mode">
    Open your browser console on Twitch and run:

    ```javascript
    localStorage.ffzDebugMode = true;
    ```

    Then reload the page. FFZ will now load from your local development server at `localhost:8000`.
  </Step>
</Steps>

## Write your first add-on

Create a new file anywhere in your project (or in a separate add-on repository) with the following minimal add-on:

```typescript addon.ts
import { Addon } from 'utilities/addon';

export default class MyFirstAddon extends Addon {

  static info = {
    id: 'my-first-addon',
    name: 'My First Add-On',
    version: '1.0.0',
    description: 'A minimal FrankerFaceZ add-on.',
    author: 'Your Name'
  };

  async onEnable() {
    // Register a setting
    this.settings.add('my-first-addon.show-hello', {
      default: true,
      ui: {
        path: 'Add-Ons > My First Add-On >> General',
        title: 'Show Hello Message',
        description: 'Display a greeting in the console when enabled.',
        component: 'setting-check-box'
      }
    });

    if (this.settings.get('my-first-addon.show-hello')) {
      this.log.info('Hello from My First Add-On!');
    }
  }

  async onDisable() {
    this.log.info('My First Add-On is disabled.');
  }
}

// Register the add-on with FrankerFaceZ
Addon.register(MyFirstAddon);
```

<Info>
The `Addon` class automatically injects `this.settings` (SettingsManager) and `this.i18n` (TranslationManager). You don't need to declare these as dependencies manually.
</Info>

## Enable the add-on developer server

To test a locally-hosted add-on, enable the add-on developer server in FFZ's settings:

1. Open the FFZ settings panel (click the FFZ logo in the Twitch navigation)
2. Navigate to **Add-Ons** in the settings
3. Enable **Use Add-On Developer Server**

This tells FFZ to also check `https://localhost:8001` for add-on files in addition to the official add-on CDN.
