---
title: "FrankerFaceZ"
description: "The root singleton class that boots and owns all FrankerFaceZ modules."
---

`FrankerFaceZ` is the top-level class that extends `Module`. Exactly one instance is created at page load, assigned to `window.ffz`, and stored as `FrankerFaceZ.instance`.

```typescript
import Module, { State } from 'utilities/module';

class FrankerFaceZ extends Module { ... }

window.FrankerFaceZ = FrankerFaceZ;
window.ffz = new FrankerFaceZ();
```

## Accessing the singleton

```typescript
// From anywhere on the page
const ffz = window.ffz;

// Via the static getter
const ffz = FrankerFaceZ.get();

// From inside an Addon or Module
const ffz = window.FrankerFaceZ.get();
```

<Note>
  `FrankerFaceZ.get()` and `window.ffz` always refer to the same object. Prefer `window.FrankerFaceZ.get()` in add-on code so the reference survives any early-load timing issues.
</Note>

## Static methods

### `FrankerFaceZ.get()`

```typescript
static get(): FrankerFaceZ
```

Returns `FrankerFaceZ.instance`. This is the canonical way to retrieve the singleton from outside the module tree.

## Instance methods

### `generateLog()`

```typescript
async generateLog(): Promise<string>
```

Iterates over every registered module, calls each module's own `generateLog()` method (if it defines one), and assembles the results into a single formatted string. Initialization log messages captured before the extension finished booting are prepended as an `initialization` section.

Each section is separated by a dashed rule:

```
initialization
-------------------------------------------------------------------------------
[0:00:01] info | core: Loaded descriptions of 42 modules.

settings
-------------------------------------------------------------------------------
{ ... }
```

### `discoverModules()`

```typescript
async discoverModules(): Promise<void>
```

Uses a webpack `require.context` to scan `src/modules/` for all top-level module files, then calls `loadFromContext()` to register each discovered module. Called automatically during construction before `enable()` is invoked.

### `enableInitialModules()`

```typescript
async enableInitialModules(): Promise<void>
```

Iterates over all registered modules and calls `enable()` on any that have `should_enable` set to `true`. Returns a `Promise` that resolves when every initial module has finished enabling.

## Startup sequence

The constructor runs the following steps synchronously and then continues asynchronously:

<Steps>
  <Step title="Assign singleton">
    `FrankerFaceZ.instance = this` and `window.ffz = this` are set.
  </Step>
  <Step title="Install extension port">
    If the page body has a `data-ffz-extension` attribute, an inter-process communication port is installed.
  </Step>
  <Step title="Inject core modules">
    `settings`, `experiments`, `i18n`, `staging`, `load_tracker`, `socket`, `pubsub`, `site`, and `addons` are injected. `vue` is registered without being required.
  </Step>
  <Step title="Discover modules">
    `discoverModules()` scans `src/modules/` via webpack context and registers all found modules.
  </Step>
  <Step title="Enable root">
    `enable()` transitions the `FrankerFaceZ` module itself to `State.Enabled`, which cascades to all `requires` dependencies.
  </Step>
  <Step title="Enable initial modules">
    `enableInitialModules()` calls `enable()` on every module that has `should_enable = true`.
  </Step>
</Steps>
